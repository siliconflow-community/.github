## Development Guidelines

### Code Style

- Follow the existing code style in the repository
- Use meaningful variable and function names
- Write clear commit messages
- Keep changes focused and atomic

### Testing

- Write tests for new functionality
- Ensure all tests pass before submitting
- Include tests for bug fixes

### Documentation

- Update README and documentation for API changes
- Add inline comments for complex logic
- Provide examples for new features

## Pull Request Process

1. **Title your PR clearly** - Describe what you're fixing or adding
2. **Reference related issues** - Link to any related issues with `fixes #123`
3. **Describe your changes** - Explain what you did and why
4. **Add screenshots** - Include screenshots for UI changes
5. **Ensure tests pass** - Run the test suite locally
6. **Keep it small** - Large PRs are harder to review
